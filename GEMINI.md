# Правила работы для Gemini CLI

## Технический стек
- **Игра:** Minecraft 1.21.1
- **Загрузчик модов:** NeoForge
- **Базовый мод:** Create (Addon)
- **Библиотеки:** WebsNodeLib (внутренняя)

## Контекст проекта
- Мод посвящен космонавтике (ракеты, двигатели, титан).
- Основной пакет: `dev.devce.rocketnautics`
- Использует Create для механик и Ponder для обучения.

## Важные архитектурные особенности (WebsNodeLib & Sputnik)
- **Десериализация WGraph / WNode (NBT):** При загрузке узлов из NBT необходимо вызывать `addNode(newNode)` ДО вызова `newNode.load(tag)`. Это нужно для того, чтобы узел успел получить ссылку на `parentGraph` и доступ к `RegistryAccess` (без этого элементы UI, такие как `WItemPicker`, не смогут корректно десериализовать предметы, и их значения будут сбрасываться).
- **Синхронизация и вычисления узлов (Evaluators):** Узлы `redstone_receiver`, `redstone_transmitter` и другие логические блоки, которые взаимодействуют с миром или сетью Create (`LinkedSignalHandler`), обязаны иметь проверку `if (sputnik.getLevel().isClientSide()) return;` внутри метода `setEvaluator`. Сеть Create Wireless Redstone существует только на сервере. Если клиентские узлы попытаются произвести эти вычисления локально, они постоянно будут перезаписывать правильные серверные сигналы нулями во время работы UI.

## Карта классов и файлов (Справочник архитектуры)

| Файл / Класс | Зона ответственности | Где инициализируется (создается) | Где используется / вызывается |
| :--- | :--- | :--- | :--- |
| **`SputnikBlockEntity`** | Главная логика Спутника: хранит граф (`WGraph`), загружает чанки, ищет двигатели, обновляет логику. | Регистрируется в `RocketBlockEntities`. | Вызывается из `SputnikBlock`, `NetworkHandler`, обновляется тиком сервера. |
| **`SputnikBlock`** | Блок Спутника в мире (перенаправляет клики, управляет состоянием в мире). | Регистрируется в `RocketBlocks`. | - |
| **`SputnikClientUI`** | Открывает клиентский интерфейс для редактирования графа. | - | Вызывается при правом клике по `SputnikBlock` на клиенте. |
| **`LinkedSignalHandler`** | Мост к `REDSTONE_LINK_NETWORK_HANDLER` мода Create (Wireless Redstone). | - | Тикается в `SputnikBlockEntity`, методы get/set используются в `RocketNodes` (Receiver/Transmitter). |
| **`RocketNodes`** | Регистрация всех возможных типов нод (Math, Logic, I/O, Redstone, Display). | Инициализируется при загрузке мода. | Вызывается при добавлении ноды в UI или загрузке из NBT (через `NodeRegistry`). |
| **`WGraph`** | Ядро логики `WebsNodeLib`. Содержит узлы (`WNode`), связи, обрабатывает их тик и NBT (сериализацию). | Внутри `SputnikBlockEntity` (`new WGraph()`). | Передается в `WNodeScreen` на клиенте, сериализуется для сети/сохранения. |
| **`WNode`** | Узел графа. Хранит пины (`WPin`), элементы UI (`WElement`) и выполняет лямбду логики (`Evaluator`). | Динамически (в UI или при загрузке NBT) через `NodeRegistry.createNode`. | Добавляется в `WGraph`, рендерится в `WNodeScreen`. |
| **`WPin`** | Входные и выходные контакты узла, передают числа `double`. | Создаются внутри регистрации ноды (через `node.addInput`/`addOutput`). | Используются в `WGraph` для передачи значений по `WConnection`. |
| **`WItemPicker`** | UI-элемент узла (в `WebsNodeLib`) для выбора ItemStack (фильтры частот). | Создается в `RocketNodes` для нод Receiver/Transmitter. | Сохраняет NBT, требует `RegistryAccess` при загрузке из графа. |
| **`SputnikNodeSyncPayload`** | Сетевой пакет для синхронизации NBT графа между клиентом и сервером. | Создается в `WNodeScreen` (клиент) при сохранении. | Обрабатывается в `NetworkHandler` на сервере для обновления `SputnikBlockEntity`. |
| **`WNodeScreen`** | Окно (GUI) редактирования графа: панорамирование, связи, элементы управления. | Создается в `SputnikClientUI`. | Рендерит узлы, связи, обрабатывает клики и сохранение в сеть. |
